      • Seperate inbound/outbound rules
      • + - NACL Limits

      • + - Stateless
        • Traffic strictly filtered
    • + - Security Group VS NACLs
      • + - SG
        • Deny all inbound traffic until allow
        • Allow rules only
        • + - Allow all outbound traffic until allow

          • By default all outbound traffic is allowed,
            but if you specificy a rule to allow somthing, everything else is denied

        • Stateful - Return traffic is allowed
      • + - NACL
        • Default rule: deny all
        • Can have permit and deny rules
        • Can apply only One NACL per subnet. Can create unlimited
        • NACL are stateless - Traffic strictly filtered

      • + - Storage Types

          • Object Based storage
          • + - Types
            • Standard
            • RRS
        • + - EBS (Elastic Block Storage)
          • + - Elastic Block Storage Characteristics
            • Billed on Storage capacity and I/O
            • Does not need to be attached to an instance
            • Can be transferred between AZ
            • EBS volumes are design for an annual failure rate (AFR) of between 0.1% - 0.2%
            • + - EBS volume data is replicated across multiple servers in an availability zone
              • 3 times to 3 different servers in the AZ
            • SLA is 99.95%
          • + - Volume Types
            • + - Image
            • + - SSD
              • + - GP-SSD (gp2)
                • Default Volume Type
              • PIOPS (Provisioned IOPS) (io1)
            • + - HDD
              • ThroughPut Optimized HDD (srt1)
              • Cold HDD (sc1)
            • Magnetics
        • + - EFS (Managed File System)
          • Network Attached Storage
          • Based on NFS4
          • Supports only Linux instances
          • File system grow or shrink as you add or remove data
    • + - Increasing IOPS Performance
      • + - Stripping/Raid Configuration
        • Multiple stripped gp2 or standard volumes (typically RAID 0)
        • Multiple stripped PIOPS volumes (typically Raid 0)
        • Function of the guest OS
      • + - ~ EBS Optimized Instances
        • Dedicated capacity for Amazon EBS I/O
        • 425Mbps - 14,000 Mbps
        • Not possible to enable on Micro instances
        • GP-SSD within 10% of baseline and burst performance 99% of the time
        • PIOPS within 10% of provisioned performance for use with all EBS volume types 99.99% of the time
        • Additional hourly fee
        • Some of the EC2 instances have EBS optimization enabled by default
  • + - EBS Snapshots and Replications
    • + - EBS Snapshots Characteristics
      • + - Point-in-time snapshots

        • Freezes the instances and make a copy immediately without regards to what is happening in that instance

          Is not a clean cutover snapshot when done on a running instance as there maybe transactions/processes running when the snapshot is requested

          Only way to ensure a clean snapshot is to power the instance down and then take a snapshot

      • Supports incremental snapshots
      • Filelevel backup and restore is not possible
      • For more granualar backup, we will need to use 3rd party solutuons
      • Billed only for the changes blocks
      • Deleting a snapshot removes only the data not needed by any other snapshots
      • EBS leverages S3 for snapshot storage
    • + - EBS Snapshots Features
      • Resizing EBS volumes
      • Sharing EBS snapshots
      • Copying EBS snapshots across regions
      • + - Lazy Loading

        • When a snapshot is recalled from S3, an initial bit set of information is recovered which allows the instance to run the snapshot. During this proces the performance of the instance/data read/write will be degraded until the complete snapshot is recalled

      • + - Pre-Warming EBS volumes
        • A Process to avoid Lazy Loading
        • + - How to Pre-Warm on Windows/Linux
          • Format the drive (new volume)
          • Use dd for Windows (Drive and data)
  • + - Working with Amazon S3
    • One of the first services launched in AWS
    • + - Amazon S3 Characteristics
      • + - It's not a file system
        • Not meant for transactional files
        • Purely storage only
        • Folders are representation and not really a folder
      • AWS Region-level storage
      • Support REST and SOAP APIs
      • Globally unqiue S3 bucket name
      • Server-side encryption of data at rest
      • + - Two types of S3 Storage
        • + - Standard Storage
          • 99.999999999% durability
          • 99.99% availability
        • + - Reduced Redundancy Storage (RRS)
          • 99.99% durability (Average annual expected loss of 0.01% of objects)
          • 99.99% availability
      • + - Amazon S3 Replication

        • S3 stores data in multiple facilities and on multiple devices within each facilities

          RRS Amazon S3 does not replicate objects as many times as standard Amazon S3

          Synchronously stores data across multiple facilities before confirming that the data has been successfully stored

          Calculates checksums on all network traffic to detect corruption of data packets

    • + - Amazon  S3 Features
      • + - Versioning
        • Ability to preserve, retreive and restore every version of every object
      • Cross-region replication (CRR)

      • Event notifications
      • Data lifecycle management
      • + - Permissions

          • Bucket level security
          • Useful for security outside the AWS environment
        • + - ACLs

  • + - Configuring Route 53
    • Named after default DNS port : 53
    • + - Route 53 Characteristics
      • Worldwide Distributed DNS
      • DNS is a Database of name to IP mappings
      • Route 53 has 100% SLA uptime
      • Route 53 has API
      • Can configure Server Health Checks in Route 53
    • + - DNS Record Types
      • A = Address record
      • + - CNAME = Canonical record name
        • Alias
        • Name that points to another name
      • MX = Mail exchange record
      • AAAA = IPV6 address record
      • + - TXT = Text record
        • A general purpose entry in DNS
      • + - PTR = Pointer Record
        • Opposite of A record
        • responds with the name in response of an IP
      • SRV = Service Locator
      • + - SPF = Sender Policy Framework
        • To prevent email spoofing
        • An SPF lists all the IP addresses of the authorized smtp server.
      • + - NS = Name server record
        • A name server that resolves the names of the resources in the domain
        • For example: to query location of 'images.' in image.florahospitality.com
      • + - SOA = Start of authority record
        • Primary name server that has all the configuration for the particular DNS zone/ Domain name
        • TTL: Time to expire current cached data
    • + - Routing Policies
      • + - Single

        • Associate multiple IP addresses to an A record and the IP addresses will be served in a round-robin fashion

          No health checks

      • + - Weighted
        • Similar to Single, but can assign a weight on one or more IPs
      • + - Latency
        • Will route to IP based on AWS's table of latency to locations around the world
      • + - Failover
        • Ability to route to an alternate IP address in case the service located in the first IP fails
      • + - Geolocation
        • Routes to an IP addres which is has locale specific content, based on source IP address
